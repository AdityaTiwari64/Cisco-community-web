    The "Signal-Sense" Network Mapper

    Description: A solution to the common problem of unreliable campus Wi-Fi and cellular service. This tool is a crowdsourcing app that allows students to map network dead zones in real-time. The app collects anonymous data on signal strength, speed, and latency at different locations and times. The AI component uses this data to create a predictive heatmap of the campus network, forecasting when and where congestion will occur. This allows students to proactively find the best spots for stable connectivity.

    Requirements:

    • A mobile application (for Android and iOS) to automatically collect network data in the background
    • A backend server and database to store and process the crowdsourced data.
    • A machine learning model to generate a predictive heatmap of network quality
    • Integration with a mapping service (like Google Maps or OpenStreetMap) to visualize the data.

    Outcomes:

    • A real-time, dynamic map of campus network quality.
    • Actionable insights for the university's IT department to improve network infrastructure
    • Improved ability for students to find reliable connectivity when they need it most

    Conditions:

    • Sufficient user adoption is critical for gathering enough data to create a useful map
    • The app must request necessary permissions (location, network state) from the user.
    • All collected data must be anonymized to protect user privacy.

    Potential Datasets:

  • This project primarily relies on generating its own dataset through crowdsourcing.
  • UCI Wireless Indoor Localization Data Set: While focused on indoor positioning, this dataset contains extensive Wi-Fi signal strength measurements (RSSI) from multiple access points. It's excellent for developing initial models to process and interpret signal data. Link to UCI Dataset
  • Kaggle WiFi Localization Dataset: This dataset contains WiFi signal strength data collected in an indoor space. It's useful for understanding how signal strength varies by location, which is a core part of this project idea. Link to Kaggle Dataset

    INTERMEDIATE LEVEL

    "Annapurna" Smart Ration & Mess Management System

    Description: : College hostels often struggle with food waste, unpredictable meal attendance, and inefficient inventory management, leading to either food shortages or wastage. IoT Solution: A smart mess system that optimizes food preparation and reduces waste.

    Key Requirements:

    • Hardware: RFID/NFC scanners at the mess entrance, IoT-enabled weight sensors for large food containers.
    • A central cloud platform to receive and process data from the IoT devices
    • A predictive AI model to forecast student attendance and food consumption.
    • A dashboard for the mess manager to view real-time attendance, inventory levels, and demand forecasts.
    • An alerting system for low inventory levels.

    Outcomes

    • A significant reduction in food waste (e.g., a 20-30% reduction).
    • Automated and optimized inventory management, preventing shortages.
    • Cost savings for the hostel administration through efficient purchasing.

    Conditions:

    • Requires initial investment in hardware and installation.
    • Reliable internet connectivity is needed for the IoT devices to function correctly
    • Integration with the existing student ID card system (if RFID/NFC enabled) is highly desirable.

    Potential-dataset

    • This system will generate its own dataset
    • For initial modeling, one could look at generic food consumption or restaurant demand forecasting datasets on platforms like Kaggle to understand patterns, though they won't be specific to a hostel environment.
    Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹40,000
