    CLOUD COMPUTING CHALLENGE - FULL DETAILS

    INTERMEDIATE LEVEL

    The "Bandwidth-Smart" Cloud Backup

    Scenario: A secure cloud backup solution designed for users with limited or unreliable internet. It uses an AI model for Adaptive Sync Scheduling, learning user patterns to perform backups during optimal off-peak hours. It utilizes delta syncing and data deduplication to minimize data transfer.

    Key Requirements:

    • A client-side application to monitor user activity, file changes, and network conditions.
    • An AI model to predict off-peak hours based on historical data
    • Efficient algorithms for delta syncing (sending only file changes) and deduplication.
    • A secure and robust cloud storage backend.

    Outcomes:

    • Reliable and automated backups even on low-bandwidth or intermittent connections.
    • Minimized impact on the user's internet performance during active hours.
    • Reduced data usage and potentially lower cloud storage costs.

    Conditions:

    • The client-side application requires system permissions to monitor files and network.
    • The predictive model's effectiveness depends on the regularity of the user's schedule and network patterns.

    Potential Datasets

    • This system will primarily generate its own operational data. For initial research, public network traffic datasets (e.g., from CAIDA) could be analyzed to understand internet congestion patterns
    Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹40,000

    CLOUD COMPUTING CHALLENGE - FULL DETAILS

    INTERMEDIATE LEVEL

    The "Campus Brain"

    Scenario: A centralized, AI-powered knowledge hub for college life. This platform solves the problem of scattered and hard-to-find information by ingesting data from various sources like official websites, student-uploaded notes (from Google Drive), and even informal chat groups. Students can ask questions in natural language (e.g., "When is the tech fest?") and get instant, accurate answers, complete with source links. It also provides a structured way to find peer-sourced advice on topics like professor reviews or project groups.

    Key Requirements:

    • A powerful, natural language search interface (likely using a Retrieval-Augmented Generation - RAG - model).
    • Data ingestion pipelines to connect to and process data from diverse sources (web scraping for websites, APIs for Google Drive, and methods to import chat histories).
    • A robust vector database to store and efficiently query the information.
    • A user-friendly front-end where students can ask questions and browse curated topics.
    • A verification or rating system for user-submitted content (like notes and advice) to ensure quality.

    Outcomes:

    • A single, reliable source of truth for all campus-related information.
    • A significant reduction in the time students spend searching for information.
    • Preservation of institutional knowledge that is often lost when senior students graduate.
    • Faster onboarding for new students.

    Conditions:

    • Requires permissions and potentially APIs to access data from different platforms.
    • Strong user adoption and contribution are key to building a comprehensive knowledge base.
    • A moderation system is essential to manage the quality and appropriateness of user-submitted content.

    Potential Datasets

    • This project's primary dataset will be the one it builds from the college's specific data sources.
    • The core AI doesn't need to be trained from scratch. You would use pre-trained Large Language Models (LLMs) as the foundation for the search and Q&A functionality. You can find many open-source models on platforms like Hugging Face.
    Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹40,000
