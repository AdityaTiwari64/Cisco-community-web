    INTERMEDIATE LEVEL

    The "FractureNet" X-Ray Assistant

    Description:A deep learning application designed to assist medical professionals by rapidly analyzing X-ray images for bone fractures. The tool generates a visual heatmap to pinpoint the fracture location, suggests the fracture type, and provides a confidence score.

    Requirements:

    • A Convolutional Neural Network (CNN) model trained on a large dataset of X-ray images.
    • Functionality to generate heatmaps (using techniques like Grad-CAM) to visualize areas of interest.
    • A user interface for doctors to upload X-ray images and view the analysis.
    • The model should be able to classify different types of fractures.

    Outcomes:

    • Reduced time for fracture diagnosis in emergency situations.
    • Increased accuracy and a reduction in diagnostic errors.
    • An effective tool for prioritizing critical cases in a busy clinical setting.

    Conditions:

    • Access to a large, well-labeled dataset of medical images is critical and often difficult to obtain.
    • The system must comply with medical data privacy regulations (e.g., HIPAA).
    • The final tool must be validated by certified radiologists before any clinical use. This tool is an assistant, not a replacement for a doctor.

    Potential Datasets:

    • MURA (Musculoskeletal Radiographs) Dataset: A large dataset of bone X-rays from Stanford University. Link to MURA Dataset
    • ChestX-ray8: While focused on chests, it is a good example of a large-scale medical imaging dataset. Link on NIH Website
    Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹40,000

    INTERMEDIATE LEVEL

    The "Vision Check AI"

    Description: An accessible and accurate vision screening tool that uses a device's camera and computer vision. The application guides a user through a standard eye test (like a digital Snellen chart) presented on their screen. The system uses real-time video processing to track the user's head position and analyze their verbal or typed responses to determine their visual acuity.

    Key Requirements:

    • A computer vision model for real-time head and eye tracking to ensure the user is at a consistent distance.
    • A calibrated UI for displaying a digital Snellen chart or equivalent vision test.
    • An algorithm to process user input (speech-to-text for verbal responses or keyboard input).
    • A procedure to guide the user in measuring their distance from the screen for accuracy.

    Outcomes:

    • Provides a preliminary, non-diagnostic estimate of visual acuity.
    • Increases accessibility to basic vision screening for individuals in remote areas.
    • A clear report that advises the user to seek a professional examination for a formal diagnosis.

    Conditions:

    • The accuracy is highly dependent on environmental factors like screen resolution, ambient lighting, and correct user distance.
    • The application must explicitly state it is a screening tool and not a substitute for a professional eye exam.

    Potential Datasets:

  • GazeCapture: An alternative large-scale dataset for gaze estimation, collected from over 1,450 people. Link to GazeCapture
  • The core logic relies on the standardized Snellen chart, not a dataset for training vision results. Link to GazeCapture
    • Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹40,000

    ADVANCED LEVEL

    The "Smart Study Hub"

    Description: A dynamic study planner that goes beyond simple calendars to solve the problem of managing scattered notes and inefficient study habits. This tool uses AI to create personalized study schedules based on a user's subjects, exam dates, and confidence levels. It features an automated note summarizer that can process uploaded notes (PDFs, images) to generate summaries and flashcards.

    Key Requirements:

    • NLP model for abstractive or extractive text summarization.
    • Optical Character Recognition (OCR) technology to extract text from images of notes.
    • A scheduling algorithm that prioritizes topics based on user-inputted confidence levels and deadlines.
    • A user-friendly interface to manage subjects, upload notes, and view the generated study plan and flashcards.

    Outcomes:

    • An optimized, adaptive, and personalized study schedule for each user.
    • Automated creation of concise summaries and flashcards, saving significant student time.
    • Improved learning outcomes by focusing revision on weaker areas.

    Conditions:

    • The quality of the summaries and text extraction is dependent on the clarity and structure of the source notes.
    • Requires active and honest input from the user regarding their confidence levels for the schedule to be effective.

    Potential Datasets:

  • For summarization: CNN/DailyMail Dataset or XSum (Extreme Summarization) are standard benchmarks. Link to CNN/DailyMail on Hugging Face Link to CNN/DailyMail on Hugging Face

    Team Size: 1 Member
    Prize Pool: ₹40,000

    AI & ML CHALLENGE - FULL DETAILS

    INTERMEDIATE LEVEL

    The "Campus Hive" (P2P Doubt Solver 2.0)

    Description: A peer-to-peer doubt-solving platform supercharged with AI to address the difficulty of getting fast, verified answers. Its competitive edge comes from an AI that routes questions to the most qualified students based on their skill profiles. It also uses a GenAI model to provide an instant "first-draft" answer.

    Key Requirements:

    • A user profiling system to track subjects, skills, and past performance.
    • An NLP-based question classification and routing model.
    • Integration with a large language model (LLM) API for generating preliminary answers.
    • A reputation system (leaderboard, badges) to incentivize quality answers.

    Outcomes:

    • Significantly reduced waiting time for getting answers to academic questions.
    • A curated community where knowledgeable students are recognized and rewarded.
    • A reliable platform for collaborative learning.

    Conditions:

    • Requires a critical mass of active users to ensure questions are answered.
    • The GenAI-generated answers must be clearly labeled as unverified drafts.
    • A moderation system is needed to handle incorrect or inappropriate content.

    Potential Datasets:

  • Stack Exchange Data Dump: Can be used to train question routing models based on tags and content. Link to Stack Exchange Data
    • Duration: 36 Hours
    Team Size: 1 Member
    Prize Pool: ₹40,000
